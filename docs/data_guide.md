# Data Guide

This document describes all datasets in the recruitGPT pipeline: what they are, how they're produced, and how they're consumed.

---

## Data Flow

```
       Synthetic raw materials                Distilled training data              Derived training data
       (Claude generates text)               (Claude generates labels)            (computed from raw materials)
       ────────────────────────              ─────────────────────────            ─────────────────────────────

       generate_jds.py                       distill_data.py                      build_embedding_pairs.py
         → data/jds/all_jds.jsonl      ───→    → data/generated/train_raw.jsonl     → data/pairs/train_triplets.jsonl
                                        │
       generate_resumes.py              │   filter_data.py                       mine_hard_negatives.py
         → data/resumes/all_resumes.jsonl     → data/generated/train_clean.jsonl    → data/pairs/hard_negatives.jsonl

                                                                                 build_reranker_data.py
                                                                                    → data/reranker/train.jsonl

                                                                                 build_graph.py
                                                                                    → data/graph/recruit_graph.gpickle
```

---

## 1. Synthetic Raw Materials

Raw text data generated by Claude. These are **inputs** to the distillation step, not training data themselves.

### 1.1 Job Descriptions

| | |
|--|--|
| **Script** | `scripts/generate_jds.py` |
| **Output** | `data/jds/all_jds.jsonl` |
| **Count** | 200–300 |
| **Cost** | ~$1–2 (Claude Sonnet) |

```jsonl
{
  "id": "jd_0001",
  "role": "Senior Backend Engineer",
  "industry": "Fintech",
  "seniority": "senior",
  "text": "We are looking for a Senior Backend Engineer to join our Payments team..."
}
```

Covers 7 role categories (backend, frontend, data, infra, product, mobile, management) x 8 seniority levels x multiple industries to ensure diversity.

### 1.2 Resumes

| | |
|--|--|
| **Script** | `scripts/generate_resumes.py` |
| **Output** | `data/resumes/all_resumes.jsonl` |
| **Count** | 200–300 |
| **Cost** | ~$1–2 (Claude Sonnet) |

```jsonl
{
  "id": "resume_0001",
  "name": "Alex Chen",
  "role_category": "backend",
  "current_title": "Senior Backend Engineer",
  "seniority": "senior",
  "years_of_experience": 7,
  "text": "## Alex Chen\nSenior Backend Engineer\n\n**Summary**\n..."
}
```

LinkedIn-style profiles with specific achievements, company names, skill lists, and education.

## 2. Distilled Training Data (Claude as Teacher)

Claude reads the raw materials and produces structured outputs. These outputs become the **labels** that the student model (Qwen3.5-0.8B) learns to imitate.

### 2.1 Query Parsing samples

| | |
|--|--|
| **Script** | `scripts/distill_data.py --tasks query_parsing` |
| **Input** | JD text |
| **Output** | Structured JSON (skills, seniority, constraints) |
| **Count** | 1,500 |
| **Cost** | ~$5 (Claude Sonnet standard) / ~$2.5 (Batch API) |
| **Used by** | `src/train.py` → fine-tune Qwen3.5-0.8B |

```jsonl
{
  "task": "query_parsing",
  "id": "qp_jd_0001_0000",
  "source_jd_id": "jd_0001",
  "messages": [
    {"role": "system", "content": "You are a recruiting intelligence system..."},
    {"role": "user", "content": "Job Description:\n---\nWe are looking for..."},
    {"role": "assistant", "content": "{\"role_title\": \"Senior Backend Engineer\", \"seniority\": \"senior\", ...}"}
  ]
}
```

The `messages` field is in chat format, ready for SFTTrainer. Loss is computed only on the last `assistant` turn.

### 2.2 Match Explanation samples (optional)

| | |
|--|--|
| **Script** | `scripts/distill_data.py --tasks match_explanation` |
| **Input** | JD text + Resume text |
| **Output** | Structured match report (Strengths / Gaps / Interview Focus / Recommendation) |
| **Count** | 1,500 |
| **Cost** | ~$16 (Claude Sonnet standard) / ~$8 (Batch API) |
| **Used by** | `src/train.py` → fine-tune Qwen3.5-0.8B (if you want a local explainer) |

```jsonl
{
  "task": "match_explanation",
  "id": "me_jd_0001_resume_0042_0000",
  "source_jd_id": "jd_0001",
  "source_resume_id": "resume_0042",
  "messages": [
    {"role": "system", "content": "You are a senior technical recruiter..."},
    {"role": "user", "content": "Job Description:\n---\n...\n---\n\nResume:\n---\n...\n---"},
    {"role": "assistant", "content": "**Strengths**\n- 6 years backend experience..."}
  ]
}
```

This task is optional for the matching pipeline. If open-recruiter only needs ranking (not explanations), skip this to save ~$8.

### 2.3 Quality filtering

| | |
|--|--|
| **Script** | `scripts/filter_data.py` |
| **Input** | `data/generated/train_raw.jsonl` |
| **Output** | `data/generated/train_clean.jsonl` |

Filters applied:

**Query Parsing:**
- Output is valid JSON
- All 8 required fields present (`role_title`, `seniority`, `required_skills`, etc.)
- `required_skills` is non-empty list
- `seniority` is a known value (intern/junior/mid/senior/staff/lead/manager/director)
- Total text length within bounds

**Match Explanation:**
- All 4 sections present (Strengths, Gaps, Interview Focus, Recommendation)
- No section is empty
- Recommendation is a single sentence

Expected pass rate: **85–95%**. Rejected samples are logged with reason codes.

## 3. Derived Training Data (computed, no API cost)

### 3.1 Embedding triplets

| | |
|--|--|
| **Script** | `scripts/build_embedding_pairs.py` |
| **Input** | JDs + Resumes + Query Parsing outputs |
| **Output** | `data/pairs/train_triplets.jsonl` |
| **Count** | 5,000–10,000 |

```jsonl
{
  "query": "Senior Backend Engineer, Go, PostgreSQL, Kafka, 5+ years, Fintech",
  "positive": "Alex Chen | Senior Backend Engineer | ByteDance... Go, Redis, Kafka...",
  "negative": "Dana Zhou | Product Manager | 3 years... Jira, roadmap planning..."
}
```

How pairs are built:
1. Use Query Parsing JSON to create a structured query string per JD
2. Match JDs to resumes by role category + seniority overlap → **positive** pairs
3. Random resumes from different categories → **easy negatives**
4. Hard negatives are mined separately (see 3.2)

### 3.2 Hard negatives

| | |
|--|--|
| **Script** | `scripts/mine_hard_negatives.py` |
| **Input** | Triplets + current BGE model |
| **Output** | `data/pairs/hard_negatives.jsonl` |

Uses the current embedding model to retrieve top-K candidates for each query, then picks results that are **high similarity but wrong match** (e.g., same skills but wrong seniority). These are the most valuable training signals for the embedding model.

### 3.3 Reranker pairs

| | |
|--|--|
| **Script** | `scripts/build_reranker_data.py` |
| **Input** | Embedding retrieval results |
| **Output** | `data/reranker/train.jsonl` |
| **Count** | 3,000–5,000 |

```jsonl
{
  "query": "Senior Backend Engineer, Go, PostgreSQL...",
  "candidate": "Alex Chen | Senior Backend Engineer...",
  "score": 0.85
}
```

Relevance scores come from:
- **1.0** — same role category + seniority match + skill overlap > 60%
- **0.5** — same role category but seniority mismatch or partial skill overlap
- **0.0** — different role category

### 3.4 Knowledge graph

| | |
|--|--|
| **Script** | `scripts/build_graph.py` |
| **Input** | JDs + Resumes + Query Parsing outputs |
| **Output** | `data/graph/recruit_graph.gpickle` |

Nodes and edges:

```
Skill ──RELATED_TO──→ Skill        (Go ↔ Rust, PostgreSQL ↔ MySQL)
Skill ──USED_IN──→ Role            (Kafka → Backend Engineer)
Company ──IN_TIER──→ Tier          (Google → tier1_tech)
Role ──NEXT_STEP──→ Role           (Senior Engineer → Staff Engineer)
Industry ──HIRES──→ Role           (Fintech → Backend Engineer)
```

Used at inference time (Stage ④) to boost candidates with adjacent skills, similar career paths, or same-tier company backgrounds.

## 5. Execution Order

```bash
# Step 1: Generate raw materials
python scripts/generate_jds.py --num 200
python scripts/generate_resumes.py --num 200

# Step 2: Distill training data (Claude teacher → student labels)
python scripts/distill_data.py --tasks query_parsing --num 1500
python scripts/distill_data.py --tasks match_explanation --num 1500  # optional

# Step 3: Quality filtering
python scripts/filter_data.py

# Step 4: Build embedding & reranker data (no API cost)
python scripts/build_embedding_pairs.py
python scripts/mine_hard_negatives.py
python scripts/build_reranker_data.py

# Step 5: Build knowledge graph
python scripts/build_graph.py

# Step 6: Train all models
python src/embedding/train_embedding.py --config configs/bge_finetune.yaml
python src/reranker/train_reranker.py --config configs/reranker_finetune.yaml
python src/train.py --config configs/qlora_qwen3_5_0_8b.yaml
```
